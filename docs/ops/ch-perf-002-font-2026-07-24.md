## 评估结论

| 问题 | 结论 |
| ---- | ---- |
| Noto + Cormorant 双 preload 能否收敛 | **否（保留双 preload）**。首页 LCP 候选含 `.editorial-hero__title`（`--font-display` / Cormorant 500）与 body Noto；去掉 display preload 会把 hero 字推到 swap 后，伤 `/` LCP。 |
| display 能否仅关键路径 | **否（本波不做）**。display 在 home / section / blog card / article / links / projects 广泛引用；关键化需 layout 拆分与 CSS 联动，越界本 wt（css-route 另波）。 |
| Noto weight | **保持 `400` + `700`**。全站 `font-weight: 700/800` 大量使用 body 面；800 由 700 合成。任务写的「400/600」不适用本设计（无 600 UI 体）。 |
| Cormorant weight | **裁到 `500` only**。全部 `--font-display` 规则为 `font-weight: 500`；`600` 未引用。 |
| JetBrains | **保持 `preload: false`**；weight **`400` + `700`**（labels/kicker 用 700；inline code `600` 近邻合成）。未引入第二字体 / 外链 `@font-face`。 |

## Preload 矩阵 before / after

| Face | Variable | Before weight | After weight | Before preload | After preload | 备注 |
| ---- | -------- | ------------- | ------------ | -------------- | ------------- | ---- |
| Noto Sans SC | `--font-noto-sans-sc` | 400, 700 | 400, 700 | **true** | **true** | body LCP |
