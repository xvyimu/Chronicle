# 任务计划：内容运营基线与收藏资产系统

## 目标

完成下一阶段第一批可上线优化：建立上线运营基线，补强个人收藏内容资产的数据结构、展示和校验能力，让后续内容扩展、外部资料采集和运营复盘有稳定入口。

## 范围

- 新增上线基线文档，记录当前 CI、部署、生产内容、性能与真实用户指标跟踪方式。
- 扩展 `data/links.json` 的可选资产元信息，保持旧数据兼容。
- 在 `/links` 页面展示收藏资产的用途、优先级和官网/原始来源状态。
- 把链接资产校验接入现有 `pnpm check:seo`，防止重复 URL、重复分类、追踪参数、空分类回归。
- 更新内容维护文档和文档总览。

## 风险与回滚

| 风险                               | 影响                        | 缓解                                     | 回滚                                                     |
| ---------------------------------- | --------------------------- | ---------------------------------------- | -------------------------------------------------------- |
| 链接数据字段扩展导致旧数据解析失败 | `/links` 和首页链接预览异常 | 字段全部可选，保留旧 schema 兼容         | 回退 `src/lib/links.ts`、`src/types/index.ts` 和数据变更 |
