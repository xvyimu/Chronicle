## 1. 执行摘要

「西江月」是一个基于 Next.js 16 App Router 的技术博客兼作品集，采用 MDX 内容驱动 + 静态生成模式，整体架构清晰、分层合理，代码风格统一，体现了较好的工程素养。项目已具备博客列表/详情、标签聚合、作品展示、关于页、RSS、评论、明暗主题、TOC、阅读进度等完整功能。

但在本次全量审查中，发现 **4 个 P0/P1 级阻断性问题**与若干 P2/P3 级技术债务，主要集中在：

- **内容乱码**：空列表占位文案出现 GBK→UTF-8 编码错乱（mojibake），直接影响用户体验。
- **RSS 链接全部 404**：RSS 生成脚本的 slug 正则与站点 slug 规则不一致，导致所有订阅链接指向不存在的页面。
- **作品集图片全部缺失**：`projects.json` 引用的 6 张项目图均不存在于 `public/`，作品页全部显示裂图。
- **Tailwind 类名失效**：多处使用 `bg-primary` / `text-primary` 等未定义的工具类，导致分页激活态、标签悬停态、回到顶部悬停态等关键交互样式静默失效。
- **死代码与编码隐患**：3 个未使用组件、多处未使用 import、`any` 类型、React 19 effect 反模式告警。

> ✅ **以上全部问题已于 2026-06-23 完成修复。** 当前项目状态详见 §1 "执行状态"。

## 3. 架构与模块职责详解

### 3.1 架构总览

项目采用 **内容驱动静态生成** 模式，核心数据流如下：

```
┌─────────────┐    构建期读取     ┌──────────────┐    generateStaticParams   ┌─────────────────┐
│ content/*.mdx│ ───────────────▶ │  src/lib/*   │ ────────────────────────▶ │  静态 HTML 页面  │
│ data/*.json  │                  │  数据访问层   │                            │  (SSG 输出)      │
└─────────────┘                  └──────────────┘                            └─────────────────┘
                                        │                                           ▲
                                        │ 内存缓存 _cache                           │ 客户端水合
                                        ▼                                           │
                                 ┌──────────────┐   'use client' 组件   ┌──────────────────┐
                                 │  PostFull[]  │ ───────────────────▶ │ 交互层（搜索/TOC │
                                 │  (含正文)     │                       │  /主题/进度/评论) │
                                 └──────────────┘                       └──────────────────┘
```

**架构评价**：

- ✅ **分层清晰**：`lib`（数据）/ `components`（视图）/ `app`（路由）三层职责明确。
- ✅ **静态优先**：所有内容页 `generateStaticParams` 预生成，性能与 SEO 友好。
- ✅ **类型完备**：`types/index.ts` 集中定义，TS strict 模式开启，编译零错误。
- ⚠️ **数据访问层耦合文件系统**：`lib/*.ts` 直接 `fs.readFileSync`，无法在客户端复用，也无法做数据源切换（如迁移到 CMS）。
- ⚠️ **缓存策略简单**：模块级 `let _cache` 变量，无失效机制，长驻服务时内容更新不可见（对纯 SSG 影响有限）。
- ⚠️ **缺少抽象层**：RSS 脚本与 `posts.ts` 重复实现 slug 解析与 frontmatter 解析，DRY 违反并已引发 bug（见 5.2）。

## 4. 数据流转与接口定义

### 4.1 内容数据模型

```typescript
// src/types/index.ts

interface PostFrontmatter {        // MDX frontmatter 契约
  title: string;                   // 必填
  description: string;             // 必填
  date: string;                    // 必填，YYYY-MM-DD
  tags: string[];
  published: boolean;              // 缺省 true
  featured: boolean;               // 缺省 false
  image?: string;
}

interface PostMeta extends PostFrontmatter {
  slug: string;                    // 文件名去 YYYY-MM- 前缀
  readingTime: string;             // "X min read"
}

interface PostFull extends PostMeta {
  content: string;                 // MDX 正文（去 frontmatter）
}

interface Project {
  id: string;
  title: string;
  description: string;
  tags: string[];
  url?: string;
  github?: string;
  image: string;                   // ⚠️ 必填但资源缺失
  featured: boolean;
  year: number;
  longDescription?: boolean;       // ⚠️ 语义错位：实为"存在详情页"标志，但无详情页路由
}

interface TagInfo {
  tag: string;                     // 原始展示名 "Next.js"
  slug: string;                    // URL slug "next-js"
  count: number;
}
```

## 5. 问题清单（按严重程度分级）

> 分级标准：**P0** 阻断/数据损坏 · **P1** 功能错误/严重影响体验 · **P2** 质量缺陷/技术债务 · **P3** 优化建议

### 5.1 P0 — 严重（需立即修复）

#### P0-1 博客空列表文案乱码（编码损坏）

- **文件**：`src/components/blog/BlogList.tsx:15`
- **现象**：空列表占位文案显示为 `鏆傛棤鏂囩珷`，实为「暂无文章」的 GBK 字节被当作 UTF-8 解读的 mojibake。文件首部还带有 BOM（`\ufeff`）。
- **影响**：当某标签下无文章、或搜索无结果时（虽然 SearchBar 有自己的空态，但 BlogList 仍可能空），用户看到乱码，损害专业度。
- **根因**：文件曾在 GBK 环境下保存或经过错误编码转换。
